# Codex 皮肤管理器 · 功能需求文档（PRD）

版本：v1.0 ｜ 日期：2026-07-16 ｜ 状态：待评审

## 1. 背景与现状问题

当前皮肤管理器（`installer/theme-manager.mjs` 注入的 "D" 按钮面板）只有一列皮肤名、三个滑杆和几个按钮，存在以下问题：

| 问题 | 说明 |
| --- | --- |
| 看不到皮肤长什么样 | "我的皮肤"列表只有名字和一个色块，没有背景缩略图；"主题设置"预览区是一块空白粉色 |
| 上传后无反馈 | 上传背景图后面板内看不到图片，只能靠身后真实窗口变化来猜 |
| 没有示例内容 | 新用户打开面板只有一个"樱花玩偶·默认"，无法直观感受"换肤能变成什么样" |
| 调节参数太抽象 | 水平/垂直/遮罩三个 0-100 滑杆，用户不知道调的是什么、调到哪里合适 |
| 配色不可编辑 | schema 里有 accent/surface/text 三色，但 UI 完全没有暴露 |
| 无组织能力 | 主题多了以后没有搜索、排序、收藏、重命名、复制 |

## 2. 竞品调研

### 2.1 调研对象与核心功能

**搜狗/百度输入法皮肤系统**（皮肤商城 + 皮肤编辑器）
- 皮肤商城：按分类（动态、明星、动漫 IP、机械键盘、DIY 等）浏览，每款皮肤有大图预览、收藏、一键应用
- 皮肤编辑器：分区自定义（背景、按键、字体、音效、动效），提供海量模板起步，而不是从空白开始
- 百度"C位识别"：AI 识别背景图中的人物主体并浮出显示——即"背景图里的角色是一等公民"
- 产出物是可分享的皮肤文件（.ssf = zip），社区生态围绕文件分发

**Wallpaper Engine**（桌面壁纸管理的事实标准）
- 库即网格：所有壁纸以缩略图网格呈现，悬停即动，点击即用
- 每张壁纸有独立可调参数（对齐、播放速度、亮度/对比度/饱和度/色相）
- 播放列表：多张壁纸定时轮换、按登录/按一天中的时间切换
- Discover/Workshop：按流派、关键词、作者浏览社区内容

**VS Code 主题系统**
- 主题选择器方向键上下即实时预览整个工作台，Esc 即还原——"预览零成本、退出零损失"
- `workbench.colorCustomizations` 支持在任何主题上做逐 token 覆盖，且可按主题名作用域化
- 主题即 JSON 文件，市场分发、本地覆写，两层结构清晰

**Telegram 主题编辑器**
- 云主题：在线编辑器改配色，保存后所有端立即生效
- 主题即小文件，频道/链接分发，一键导入

**主题管理 UX 通用最佳实践**
- 深浅双模式都要校准全部颜色（含 accent/danger），不能只换背景
- 文本对比度至少 4.5:1（WCAG），提供对比度检查而不是让用户凭感觉
- 强调色避免高饱和，暗背景下用去饱和色调
- 用户选择需要持久化，切换需要可即时预览

### 2.2 结论：五条可借鉴的设计原则

1. **缩略图网格是主题库的标准形态**——名字列表不及格（搜狗商城、Wallpaper Engine、VS Code 扩展市场全部如此）。
2. **模板/预设起步，而不是空白起步**——内置 3-6 套完整示例主题，用户先"选"再"调"。
3. **实时预览 + 无损退出**——预览成本必须为零（VS Code 方向键模式）；本项目管理器就运行在真实 Codex 窗口内，天然具备"所见即所得"，要把它变成显式的"试用/应用/还原"心智。
4. **参数可视化**——位置调节应该是"在缩略图上拖拽"，而不是两个 0-100 滑杆；本项目已有 `BACKGROUND_SPEC.md` 构图安全区，应该画成参考线叠加在预览图上。
5. **主题即文件**——已有 `theme.schema.json` 可移植格式，这是对的，继续强化导入导出/拖拽分享。

## 3. 用户与场景

- **轻度用户**：只想换一张自己喜欢的背景图，30 秒内完成，不关心参数。→ 需要：预设主题一键套用、上传即所得。
- **调校用户**：对可读性敏感，想微调位置/遮罩/配色让代码区舒服。→ 需要：可视化编辑器、对比度提示、安全区参考线。
- **创作/分享用户**：制作主题分享给别人（对标输入法皮肤作者）。→ 需要：完整编辑能力、导出规范主题包、导入体验顺滑。

## 4. 功能需求

优先级说明：P0 = 本次优化必做；P1 = 第二阶段；P2 = 远期/可选。

### 4.1 主题库（我的皮肤）

| 编号 | 需求 | 优先级 | 说明 |
| --- | --- | --- | --- |

| LIB-2 | 内置示例主题 ≥3 套 | P0 | 除樱花默认外，内置 2-3 套完整主题（含背景图、配色、布局），首次打开即可一键切换感受效果；内置主题不可删除、可"另存为副本" |
| LIB-3 | 悬停快捷操作 | P0 | 卡片悬停出现：应用 / 编辑 / 复制 / 导出 / 删除（内置主题隐藏删除） |
| LIB-4 | 重命名 | P1 | 卡片内联重命名 |

| LIB-6 | 收藏置顶 | P2 | 星标主题排在最前 |
| LIB-7 | 空状态引导 | P0 | 主题库只有内置项时，展示"上传一张背景图，30 秒做一套皮肤"的引导入口 |

## 5. 非功能需求

- **零依赖注入**：管理器最终产物仍是单个自包含 JS 字符串，经 CDP `Page.addScriptToEvaluateOnNewDocument` 注入；不引入运行时框架，不发起外部网络请求（安全模型不变：loopback-only、no-ASAR、资产 data URL 内嵌）。
- **性能**：注入脚本 gzip 后 ≤ 150KB（不含内置主题图）；内置示例主题背景用 WebP q88 且单张 ≤ 400KB；打开面板首帧 < 100ms；滑杆/拖拽调节节流至 ≤ 60Hz 写样式、≤ 2Hz 写存储。
- **存储**：继续使用 IndexedDB（`codex-doll-skin-library`），主题总量软上限 50 套，超限提示清理；localStorage 只存轻量状态（当前主题 id、暂停位）。
- **兼容与降级**：注入的元素 id（`codex-doll-skin-runtime` / `codex-doll-theme-override` / `codex-doll-skin-menu` / `codex-doll-skin-manager`）保持不变，保证 CLI 与测试兼容；宠物/avatar-overlay 窗口继续排除在注入之外。
- **数据迁移**：schema 升级到 v2（新增 filters、多背景等字段）时，v1 主题自动无损升级，导出时可选导出 v1 兼容格式。

```jsonc
  "schemaVersion": 2,

}
```

## 7. 验收标准（P0 范围）

1. 新用户首次打开面板：能看到 ≥3 套带缩略图的示例主题，点击任一套 1 秒内全窗口生效。
2. 上传一张图：面板内立即出现该图预览，可拖拽定位、可开关安全区参考线、拖遮罩滑杆时预览与真实窗口同步变化。
3. 编辑后点"还原"：窗口回到编辑前状态，不留任何残余样式。
4. 修改三个颜色值：真实窗口的强调色/表面色/文字色随之变化，非法 hex 被拒绝。
5. 导出再导入同一主题：出现导入确认预览，入库后与原主题渲染一致。
6. `npm run skin:status` 输出的当前状态与面板显示一致；`npm test` 全绿。
